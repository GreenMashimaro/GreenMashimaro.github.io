服务器ssh配置

  1. [service]，在.ssh下，添加【authorized_keys】文件，复制本地pub【公钥】到【authorized_keys】中
  2. [service]，升级【authorized_keys】文件权限：chmod 600 authorized_keys
  3. [service]ssh重启，sudo service ssh restart

iptables

  1. 更新ubuntu：sudo apt-get update && sudo apt-get upgrade

  2. 清除iptables规则：sudo iptables -F

  3. 创建iptables配置文件：sudo vi /etc/iptables.up.rules

  4. 载入iptables配置文件：sudo iptables-restore < /etc/iptables.up.rules

  5. 查看防火墙是否启动：sudo ufw status

  6. 启动防火墙：sudo ufw enable

  7. 创建自启脚本：sudo vi /etc/network/if-up.d/iptables

  8. 给自启脚本升级权限： sudo chmod +x /etc/network/if-up.d/iptables

#fail2ban

  1. 安装fail2ban：sudo apt-get install fail2ban

  2. 打开配置文件：sudo vi /etc/fail2ban/jail.conf

  3. 查看fail2ban是否运行：sudo service fail2ban status

  4. 停止fail2ban：sudo service fail2ban stop

  5. 启动fail2ban：sudo service fail2ban start
